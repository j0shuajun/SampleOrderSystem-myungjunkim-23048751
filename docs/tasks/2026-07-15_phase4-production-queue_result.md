# Phase 4: 생산 큐와 재고 부족 처리 — 결과

Written at (KST): 2026-07-15 15:30

## 1. 한눈에 보는 요약

재고 부족으로 `PRODUCING`이 된 주문은 이제 FIFO 생산 큐에 실제 작업으로 등록되고,
계획 생산량은 `ceil(부족분 / 수율)`로 자동 계산된다.

## 2. 왜 필요했는지

Phase 3까지는 "부족하니 생산해야 한다"는 상태만 표시할 뿐, 실제로 몇 개를 만들어야
하는지·어떤 순서로 만들지가 없었다. PDF의 생산라인 화면(부족분→실생산량, 대기 큐)을
구현하려면 이 계산과 큐가 먼저 있어야 한다.

## 3. 예시로 보는 결과

시료 `S-003`(수율 0.92, 재고 70)에 200개 주문이 들어와 승인하면:

```
가용재고 70 (다른 미출고 주문 없음) -> 부족분 = 200 - 70 = 130
계획생산량 = ceil(130 / 0.92) = 142
```

승인 후 생산 큐를 조회하면 이 작업 1건이 나타난다. 서로 다른 시료·수율의 작업
여러 개를 순서대로 등록해도(`ORD-1`→16개 부족/수율0.8→20개 계획, `ORD-2`→7개
부족/수율0.5→14개 계획) 큐는 항상 넣은 순서(FIFO)대로 반환됨을 확인했다.
