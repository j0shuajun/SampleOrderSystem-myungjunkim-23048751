# Phase 9: 콘솔 UI 통합 — 결과 (마지막 Phase)

Written at (KST): 2026-07-15 20:00

## 1. 한눈에 보는 요약

`python3 main.py [경로]`로 실행하면 SPEC.md 6+1 메뉴가 뜨고, 시료 등록부터 주문
접수·승인·생산·출고·모니터링까지 콘솔에서 끝까지 진행할 수 있다. 종료 시 저장,
재실행 시 복원도 실제로 동작한다. 이로써 8개 Phase의 도메인 로직이 하나의 실행
가능한 시스템으로 완결됐다.

## 2. 왜 필요했는지

지금까지는 pytest로만 동작을 확인할 수 있었다. PRD 7장의 성공 기준("담당자가
콘솔 메뉴만으로 주문 접수부터 출고까지 진행할 수 있다")을 실제로 사람이 눌러보며
확인할 수 있게 만드는 게 이 프로젝트의 최종 목표였다.

## 3. 예시로 보는 결과 (실제로 실행해 확인한 값)

시료 `S-001`(평균생산시간5, 수율0.9, 재고3)에 10개 주문:

```
접수 완료: ORD-20260715-0001 RESERVED
승인 → 재고 부족(3<10) → 생산 큐 등록: JOB-0001 (부족분7, 계획수량 ceil(7/0.9)=8)
생산 시작 → (시간 경과 후) 완료 처리 → 재고 +floor(8*0.9)=+7 → 재고 10, 주문 CONFIRMED
출고 처리 → 재고 10-10=0, 주문 RELEASE
종료 → 저장
```

재실행하면(3번째 실행) 모니터링 메뉴에 `RELEASE 1`, `S-001 재고 0 상태 고갈`이
그대로 나타남 — 저장된 JSON과 정확히 일치.

잘못된 입력(수량에 문자 입력, 존재하지 않는 주문 ID로 승인 시도)도 프로그램이
죽지 않고 오류 메시지 후 메뉴로 돌아간다.

## 7. 제약/후속

- `seed.py`(더미데이터 생성)는 SPEC 메뉴 표에 없어 이번 범위에서 제외했다.
  필요하면 DummyDataGenerator PoC를 참고해 별도 후속 작업으로 추가할 수 있다.
- 여러 프로세스 동시 접근(파일 잠금)은 다루지 않는다(PRD 범위 제외와 일치).
- CLI 자동 테스트는 순수 함수 단위로 한정하고, 대화형 흐름은 수동/에이전트
  시나리오로 검증했다(PLAN.md에 이미 정해진 전략).

## 8. Lessons

- doc-consistency-verifier가 이번에도 실제로 값을 냈다 — Phase 9 착수 전
  "저장/복원을 언제 연결할지"와 "파일 경로를 어떻게 받을지"가 문서에 빠져있던 걸
  잡아내서, 구현자가 임의로 넘겨짚지 않게 했다.
- CLI처럼 자동 테스트로 완전히 덮기 어려운 계층은, "순수 함수는 자동 테스트 +
  대화형 흐름은 에이전트가 실제로 구동"하는 조합이 효과적이었다 — test-verifier가
  세 번 실행(정상흐름/시간경과 시뮬레이션/재실행복원)하며 계획 문서의 예시 숫자와
  실제 출력을 하나하나 대조한 게 이번 프로젝트 전체에서 가장 꼼꼼한 검증이었다.
- 프로젝트 전체를 통틀어, plan 문서에 구체적인 예시 숫자(실제 ID, 수량, 계산
  결과)를 미리 적어두는 습관이 결과 검증 단계에서 "숫자가 맞는지"를 바로 대조할
  수 있게 해줘서 반복적으로 유용했다.
