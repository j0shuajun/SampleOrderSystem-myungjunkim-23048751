# PRD: Semiconductor Sample Order System

## 1. 배경

가상의 반도체 회사 S-Semi는 연구소, 팹리스 업체, 대학 연구실 등에 반도체 시료를 납품한다.
주문량이 늘어나면서 엑셀과 메모장만으로는 주문 상태, 재고, 생산 진행 상황을 안정적으로
관리하기 어려워졌다.

운영자는 다음 질문에 빠르게 답할 수 있어야 한다.

- 이 주문은 접수만 된 상태인가, 승인된 상태인가?
- 재고가 충분해서 바로 출고할 수 있는가?
- 재고가 부족하다면 생산 라인에 들어갔는가?
- 생산이 끝나면 어떤 주문을 출고해야 하는가?

## 2. 목표

콘솔 기반의 반도체 시료 생산주문관리 시스템을 만들어 시료, 주문, 재고, 생산, 출고 상태를
한 흐름으로 관리한다.

## 3. 주요 사용자

| 사용자 | 관심사 |
|---|---|
| 주문 담당자 | 고객 주문 접수, 주문 목록 확인 |
| 생산 담당자 | 주문 승인/거절, 재고 부족 주문의 생산 처리 |
| 운영 담당자 | 상태별 주문 수, 시료별 재고 현황 확인 |

## 4. 핵심 사용자 가치

- 주문 상태를 일관된 흐름으로 관리한다.
- 충분한 재고가 있으면 불필요한 생산을 막는다.
- 부족한 수량만 생산 큐에 넣어 생산 계획을 명확히 한다.
- 거절된 주문은 정상 운영 지표에서 제외해 실제 처리 대상을 선명하게 보여준다.
- 저장소를 통해 프로그램 재실행 후에도 업무 상태를 유지한다.

## 5. 대표 사용자 흐름

### 흐름 A: 재고가 충분한 주문

1. 주문 담당자가 고객 주문을 접수한다.
2. 주문 상태는 `RESERVED`가 된다.
3. 생산 담당자가 주문을 승인한다.
4. 시스템이 재고를 확인한다.
5. 재고가 충분하면 주문 상태가 `CONFIRMED`가 된다.
6. 출고 담당자가 출고 처리한다.
7. 주문 상태가 `RELEASE`가 되고 재고가 차감된다.

### 흐름 B: 재고가 부족한 주문

1. 주문 담당자가 고객 주문을 접수한다.
2. 생산 담당자가 주문을 승인한다.
3. 시스템이 재고 부족을 확인한다.
4. 주문 상태가 `PRODUCING`이 되고 생산 큐에 들어간다.
5. 생산 라인이 FIFO 순서로 작업한다.
6. 생산 완료 후 재고가 증가하고 주문 상태가 `CONFIRMED`가 된다.
7. 출고 처리 후 주문 상태가 `RELEASE`가 된다.

### 흐름 C: 거절된 주문

1. 주문이 `RESERVED` 상태로 접수된다.
2. 생산 담당자가 거절한다.
3. 주문 상태가 `REJECTED`가 된다.
4. 모니터링의 정상 처리 흐름에서는 제외된다.

## 6. 범위

포함:

- 시료 등록, 조회, 검색
- 주문 접수, 승인, 거절
- 재고 기반 주문 처리
- 생산 큐와 생산 완료 처리
- 출고 처리
- 상태별 주문 모니터링
- 시료별 재고 모니터링
- 데이터 영속성
- 더미 데이터 또는 seed 데이터 활용

제외:

- 웹 UI
- 외부 DB 필수 의존
- 실제 장비 연동
- 사용자 인증/권한
- 복잡한 일정 최적화
- 실시간 멀티 프로세스 동시 처리

## 7. 성공 기준

- 담당자가 콘솔 메뉴만으로 주문 접수부터 출고까지 진행할 수 있다.
- 주문 상태가 명세된 상태 흐름을 벗어나지 않는다.
- 재고 충분/부족 판단이 자동으로 이루어진다.
- 생산 큐가 FIFO로 동작한다.
- 데이터가 재실행 후 유지된다.
- 테스트로 주요 상태 전이와 생산량 계산이 검증된다.
